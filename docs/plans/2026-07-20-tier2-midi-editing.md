# Tier 2: MIDI打ち込み（MIDIトラック・GM音源・ピアノロール）

## 概要・やりたいこと

- MIDIリージョンを作り、トラックごとに楽器（ドラム・ピアノ等）を割り当てて鳴らせるようにする
- ピアノロールでのマウス打ち込みで、伴奏・ドラム等のスケッチを完結させる
- 音はスケッチ品質でOK。macOS内蔵GM音源（DLSMusicDevice）を固定ホストし、汎用AUホスティングは次Tier以降に送る

## 前提・わかっていること

`/dig` での決定事項:

- **音源**: DLSMusicDevice（macOS内蔵GM音源、128音色＋ドラムキット）を1トラック=1インスタンスでホスト。AUホスティング・外部MIDI出力は非スコープ
- **トラックモデル**: オーディオ/MIDIを別種別に（Logic方式）。トラック作成時に種別を選ぶ。録音ボタン・録音機能はオーディオトラックのみ
- **入力**: マウス打ち込みのみ。MIDIキーボード録音・MIDIデバイス管理は非スコープ
- **ピアノロール**: MIDIリージョンをダブルクリックでメインウィンドウ下部ペインに開閉
- **リージョン操作**: 作成・削除・ドラッグ移動・端ドラッグでリサイズ・⌥ドラッグ複製。分割・結合は見送り
- **ノート編集**: 作成・削除・移動（ピッチ＋時間）・長さ変更・グリッドスナップ＋複数選択（ラバーバンド/Shiftクリック）一括操作＋ベロシティ編集＋コピペ（⌘C/⌘V、再生ヘッド位置に貼り付け）＋キー操作ピッチ移動（↑↓=半音、⌥↑↓=オクターブ、Logic準拠）
- **グリッド**: ピアノロールは1/32＋3連符グリッドが要件（メモリ `grid-resolution-policy` 参照。タイムラインの1/16上限とは別）
- **補助機能**: ノート作成/選択/ドラッグ中ピッチ変更時のプレビュー発音、ドラムキット選択時のGMドラムマップ名表示（Kick/Snare等）
- **Undo/Redo**: ⌘Z/⇧⌘Z。対象は構造編集のみ（ノート・リージョン・トラック追加/削除/リネーム・録音=クリップ追加）。音量・ミュート・ソロ・再生位置・選択状態は対象外（Logicもミキサー操作は別扱い）
- **保存形式**: project.json埋め込み（ノート列 pitch/start/length/velocity をリージョンごとにJSON配列で）。DAWのデファクトどおり内部形式＝プロジェクトファイル、SMFは交換用という分業。SMFエクスポートは今回見送り（JUCEの`MidiFile`で後から小規模に追加可能）
- **楽器リスト（厳選8種・仮確定）**: Piano / Electric Piano / Finger Bass / Synth Bass / Strings / Synth Lead / Synth Pad / Drum Kit。トラックヘッダのドロップダウンで選択。GMプログラム番号: Piano=0, E.Piano=4, Finger Bass=33, Synth Bass=38, Strings=48, Synth Lead=80, Synth Pad=88, Drum Kit=ch10（プログラム番号でなくMIDIチャンネル10）
- **BPM・拍子**: 既存どおり曲中固定BPM・4/4をそのまま適用
- **ミュート・ソロ・音量**: MIDIトラックにも既存の`TrackParams`をそのまま適用

技術的な前提:

- 既存構成: `Source/audio/`（オーディオスレッド）・`Source/ui/`（メッセージスレッド）・`Source/shared/`（受け渡し）。境界は`Project::buildSnapshot()`→`PlaybackSnapshot`
- MIDIノートの実体はメッセージスレッド所有のモデルに置き、スナップショット経由でオーディオスレッドへ渡す既存方針を踏襲
- DLSMusicDeviceはAudioUnit（`kAudioUnitType_MusicDevice` / `kAudioUnitSubType_DLSSynth`）。JUCEの`AudioUnitPluginFormat`でインスタンス化できる見込みだが、Phase 1冒頭で実機検証する（`juce_audio_processors`モジュール＋`JUCE_PLUGINHOST_AU=1`が必要）
- ノートのタイミングはサンプル精度でオーディオスレッドからAUに`MidiBuffer`で渡す（ブロック内オフセット付与）
- Undoは「編集前のProjectモデルのスナップショットを積む」方式。`Clip::audio`が`shared_ptr`なのでバッファ実体は共有され、構造のみのコピーで安価
- 現行Tier 1にループ再生機能は無い。ノートオフ処理の考慮対象は「シーク・再生開始・停止」のみ（ループは対象外）

レビューを受けて確定した設計方針:

- **ID設計**: `Track` / `MidiRegion` / `MidiNote` に永続化する安定ID（プロジェクト内で単調増加するuint64連番、project.jsonに保存）を持たせる。トラック削除・並び替え・AU割当・複数選択はインデックスでなくIDで扱う
- **PPQ仕様の固定**: 1四分音符 = 960 tick（1/32三連符 = 80 tickで整数表現可）。ノートの `startPpq` はリージョン相対で0以上、`lengthPpq` は1以上（リサイズ最小値も1 tick）。リージョン外へのノート移動・リサイズはリージョン範囲内にクランプする。リージョン端を越えて伸びたノートは再生時にリージョン境界でノートオフ（リージョンがマスクとして働く。Logic同様）
- **AUの所有権と寿命**: `SnapshotExchange` はスナップショットの寿命しか保証しないため、AU本体の寿命は別途管理する。PlaybackEngine（メッセージスレッド側管理部）が `TrackId → AUインスタンス` のスロットを所有し、スナップショットには非所有ポインタ＋TrackIdを載せる。AUの生成・破棄はメッセージスレッドのみ。破棄は即時に行わず「退役リスト」に入れ、スナップショットに単調増加の世代番号を付け、**そのAUを参照しうる世代のスナップショットがすべてretired経由で回収されたことを確認してから** delete する（`deleteRetired()` と同じTimerで駆動）。オーディオスレッドでの `shared_ptr` コピーは参照カウント操作が入るため使わない
- **MidiBuffer等のリアルタイム安全性**: callback内でのメモリ確保を禁止するため、`prepareToPlay()` で全バッファを事前確保する。上限を固定: 最大MIDIトラック数50（CLAUDE.mdの想定に合わせる）・1ブロックあたり1トラック最大1024イベントで `MidiBuffer::ensureSize()`、AUレンダリング用 `AudioBuffer` もトラック数分事前確保。上限超過時の切り捨て規則: **既に送ったノートオンに対応するノートオフを最優先**し、上限到達後は新規ノートオンを捨てる（捨てたノートオンに対応するノートオフも予約しない）。これで「鳴りっぱなし」を構造的に排除する。切り捨て発生はatomicフラグでUIに通知してデバッグ表示
- **ミュート/ソロとMIDI処理**: ミュート・非ソロのトラックでもMIDIイベント送信とAUレンダリングは常に行い、ミュート/ソロは**最終ミックスへの加算ゲインを0にするだけ**とする。イベント送信をスキップするとノートオフを取り逃して鳴りっぱなしになり、解除時に位置ずれした音が出るため
- **シーク時の持続音**: シーク・再生開始時は All Notes Off を送った後、再生位置を跨いでいる（noteOn < 再生位置 < noteOff の）ノートを offset 0 で再発音する。停止時は All Notes Off のみ
- **UndoとWAV GCの整合**: 保存時のWAV GC（`Project::save` の `clip-*.wav` 掃除）は「現モデル＋undo/redoスタック内の全スナップショットが参照するファイル」の和集合を参照集合として使う。undo履歴に残っているクリップのWAVは消さない（redoでWAV欠損する事故を防ぐ）
- **プレビュー発音の経路**: UIからAUを直接叩くのはスレッド境界違反。`juce::AbstractFifo` ベースのロックフリーなプレビューコマンドFIFO（UI→audio）を `shared/` に追加し、noteOn(trackId, pitch, velocity) / allNotesOff(trackId) を流す。ノートオフはオーディオ側が固定発音長（例: 0.5秒）をサンプルカウントして自動送出（UIタイマー任せにしない）。ペインを閉じた時はallNotesOffで打ち消す。**プレビューは停止中のみ有効**（再生中はUI側でプレビュー発音を発行しない・再生開始時にオーディオ側もプレビュー中ノートを打ち切る）: 再生と同一AUを共有するため、再生中にプレビュー由来のAll Notes Offを送ると伴奏まで止まる衝突を回避する
- **楽器変更**: All Notes Off → プログラムチェンジの順で送る。Drum Kit はプログラムチェンジでなくMIDIチャンネル10固定で扱う
- **テスト**: Debugビルドのassertでなく、CTestで実行できるコンソールテストターゲットを追加する（モデル・保存/読込はGUIなしで検証可能）

## ログ

### 試したこと・わかったこと
- DLSスパイク成功: JUCE 8.0.9の`AudioUnitPluginFormat::findAllTypesForFile("AudioUnit:Synths/aumu,dls ,appl")`→`createInstanceFromDescription`（同期版）でインスタンス化でき、ノートオン後のオフラインレンダリングでpeak 0.11を確認。生のAudioUnit APIへのフォールバックは不要。コンソールテストでは`ScopedJuceInitialiser_GUI`でMessageManagerの初期化が必要
- DLSMusicDeviceは出力バス2本・計4chを報告し、`disableNonMainBuses()`でも減らせない。`processBlock`には4ch分のバッファを渡さないとチャンネル範囲外アクセスでassert。ミックスはメインバス（ch0/1）のみ使用（GOTCHAS.mdにも追記）
- PlaybackEngineはGUIなしでテスト可能: `TransportState`＋`SnapshotExchange`を直接作り、`process()`に`AudioSourceChannelInfo`を渡してブロックを回せば、再生・シーク・停止・ミュートの全経路をCTestで検証できる
- JUCEのPopupMenu項目はAppleScriptのAXクリックでは押せない（`entire contents`で`menu item`は見えるが参照解決に失敗する）。CGEventの座標クリック（sandbox無効時）なら効く。マウス合成でリージョンのドラッグ操作も検証でき、pass/fail判定は保存後のproject.jsonの値で裏取りするのが確実（スクショの目視だけだと合成イベントの乱れと区別できない）
- 合成イベントを短時間に連続して撃つと、OSのクリック集約で意図しないダブルクリック扱いになることがある（リージョンが余分に作成された）。操作ごとに時間を空けて単独で検証する
- GUI検証の途中でディスプレイがスリープし、`AudioDeviceStart`（CoreAudio）内のmutex待ちでプロジェクトオープンがハングする事象に遭遇（自作コードの全スレッドはアイドルで無関係と`sample`で確認）。再起動で解消。Phase 4/5のピアノロールGUI目視確認は人間の動作確認に委ねた

### 試したこと・わかったこと（動作確認後の修正）
- ユーザー確認で「音符が無い場所で音が鳴る」報告 → 原因は自動検証が`cli-midi-test`に残したリージョン残骸（空リージョン2つ＋ノート入り複製が重なって不可視）。project.jsonを掃除して解消
- あわせて⌥ドラッグ複製を「mouseDown時に複製」から「最初にドラッグが動いた時点で複製」に修正。旧実装は⌥クリックしただけで元と完全に重なる不可視の複製が残る罠だった

### 方針変更
- AU寿命管理を世代番号方式から「スナップショットによるshared_ptr共有所有」に変更（Phase 2）。既存の`ClipPlayback::audio`と同一パターンで、shared_ptrの参照カウント操作（構築＝`buildSnapshot`・破棄＝`deleteRetired`/`push`）は全てメッセージスレッドで起きる。オーディオスレッドはスナップショット内の参照を辿るだけでコピーしない。世代番号の追跡が丸ごと不要になり、退役の安全性は既存のSnapshotExchange機構がそのまま保証する
- 楽器変更は「All Notes Off＋プログラムチェンジ」でなく「AUインスタンスの差し替え」で実現（Phase 2）。プログラムチェンジは新インスタンス生成直後（スナップショットに載せる前＝メッセージスレッドが独占所有している間）に1ブロック流して適用する。旧インスタンスは鳴っていたノートごと破棄されるためAll Notes Offが不要になり、オーディオスレッド側に楽器変更の概念が存在しなくなる
