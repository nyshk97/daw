# Tier 1: マルチトラック・タイムラインDAW

## 概要・やりたいこと

Tier 0（1トラック録音・再生・ライブ波形）を、実用最低限のマルチトラックDAWに作り替える。

- 複数トラックを作れて、各トラックでそれぞれ録音できる
- 録音開始位置を小節単位で選べる（例: 8小節目から録音）
- 小節ルーラー＋再生ヘッド（シークバー）で再生位置を表示・クリックで移動
- トラックごとのミュート・ソロ・音量
- プロジェクトの保存・読み込み

用途はボーカルの多重録音（ハモリ・ダブル等）。楽器は弾かない。

### 将来拡張の考慮（実装はしない、設計で塞がないだけ）

- **ゆくゆくビートメイク（MIDIトラック）を扱いたい。トラック数は最大50程度になりうる**
- 対応方針:
  - project.json のトラックに `"type": "audio"` フィールドを持たせる（MIDI追加時のスキーマ拡張余地）。JSONに `"version"` フィールドも持たせる
  - エンジンは「トラック＝1ブロック分のバッファを生産するもの → ゲイン/ミュート適用 → 全トラック加算」という構造にし、将来のMIDIトラックは新しい生産者を足すだけにする。**ただしMIDI用の抽象基底クラス等を先回りして作らない**（Tier 0方針の「抽象化のしすぎに注意」を維持）
  - トラックリストUIは最初から `juce::Viewport` でスクロール可能にする（50行でも破綻しない）
  - CLAUDE.md の非目的「大量トラック運用（数十〜数百トラック）は想定しない」は「〜50トラックまでは想定、数百は想定しない」に修正する

### 技術方針

- **再生はサンプル位置ベースの自前ミックスエンジン**。`AudioTransportSource` / `AudioFormatReaderSource` は廃止（1ファイル頭から再生専用の仕組みのため）
- **クリップ音声は全てメモリ常駐**（float モノラル）。試算: 20トラック×5分×48kHz×4B ≈ 1.15GB（通常時 約430MB）で許容。将来50オーディオトラック満載（≈2.9GB）が現実になったら遅延ロード/ストリーミングに移行する（今回はやらない）
- **エンジンへのクリップ/トラック構成の受け渡しはスナップショット差し替え方式**: メッセージスレッドが再生用データ（クリップのバッファポインタ・開始サンプル・トラック設定の一覧）を構築し、atomicポインタで差し替える。**旧スナップショットの解放は必ずメッセージスレッド側で行う**（GOTCHAS.md パターン3）。音量・ミュート・ソロ・BPMなど単一値は従来どおり atomic（パターン1）
- 録音は既存の Recorder（パターン4）を流用。録音停止時にWAVをメモリロードしてクリップ化する
- Tier 0 のライブスクロール波形（WaveformFifo + WaveformDisplay）はPhase 2で役目を終えて削除する

## ログ
### 試したこと・わかったこと
- スナップショット差し替えは pending/retired/current の3スロット式（`shared/PlaybackSnapshot.h` の `SnapshotExchange`）。retiredが空のときだけオーディオスレッドが差し替えるので、解放は必ずメッセージスレッドに戻る
- 音量・ミュート・ソロは `TrackParams`（atomicの束）を Track モデルとスナップショットが shared_ptr で共有する形にした。値変更でスナップショット再構築が不要になる
- カウントインは「再生ヘッドを punchIn − 1小節 に置き、punchIn 以降のサンプルだけ Recorder に書く」方式。1小節目からの録音ではヘッドが負の位置から走る（クリップは位置0以降のみなので無音、クリックだけ鳴る）
- 録音の実測: 7秒録音で WAV は4.74秒（= 7秒 − カウントイン1小節2.4秒@BPM100）。モノラル24bit・48kHzを `afinfo` で確認
- JUCEアプリはAppleScriptの合成キーストローク・`click at`（座標クリック）が効かない。ボタンのAXPress（`click button "名前"`）は効くので、CLI検証はAXPress＋スクショ＋成果物（WAV/json）確認で組む（詳細はVERIFY.md）
- 文字列リテラルに非ASCIIを含めるときは必ず `juce::String::fromUTF8(u8"...")` を通す。`"daw — "`（em-dash）を素の `const char*` 連結にしていてタイトルが「daw â」に化けた（修正済み）

### 方針変更
- Phase 1〜4 を一括で実装した（Phase 1時点の「固定WAV上書き録音の維持」「Phase 2の暫定作業フォルダ」という中間状態は作らず、最初からプロジェクトフォルダ＋クリップ録音で実装）。各Phaseで動くアプリを保つ意図はビルド・起動確認を細かく挟むことで代替
- クリップ削除時にWAVを即削除せず、保存時に未参照 `clip-*.wav` をGCする方式にした。未保存のまま終了→再読込してもクリップが欠損しないようにするため
- トラックヘッダ列は自前Viewportを持たせず、タイムラインのViewportとY座標同期（ヘッダ上のホイールは転送）にした
