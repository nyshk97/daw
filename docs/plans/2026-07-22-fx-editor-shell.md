# FXパネルの器（スライス2: 左=概要パネル常設＋下部=詳細エディタの2段構成・ON/OFF・ミキサー連動）

## 概要・やりたいこと

- ミキサー・FX領域（全6スライス）の**スライス2**。選択チャンネルの固定FXチェーンを表示する**FXパネル（左=概要・常設）＋FX詳細エディタ（下部・スロットクリックで開く）**の器を作る
- FXのDSP・エディタ中身は後続スライス（3=EQ / 4=Comp / 5=Reverb+Delay+Limiter / 6=外部AU）。今回は**区画のガワ＋ON/OFFの永続化＋チャンネル切替の配線**まで
- 全体設計はメモリ `mixer-fx-ui-direction` とスライス1のplan（2026-07-22-mixer-foundation.md）を参照

### スコープ外（後続スライス）

- FXのDSPとエディタ中身（EQカーブ・Compノブ等）→ スライス3〜5
- Ext（外部AUホスティング・レイテンシ補償）→ スライス6
- バス/MasterのFXパラメータ・ON/OFF永続化 → スライス5

### 技術的な前提（コード調査済み）

- **下部スロットの現行実装**: `MainComponent::resized()` が下部をピアノロールに割り当てている。**FX詳細（`FxDetailView`）が同じ下部スロットを使い**、`resized()` でどちらか一方だけを配置する。排他は「開く側が相手を閉じる」: `openPianoRoll()` で詳細を閉じ、詳細を開くときは `closePianoRoll()` を呼ぶ（プレビュー発音の打ち消しも既存の `closePianoRoll()` が行う）。高さは両パネル共通の `bottomPanelHeight`（既定320px）で、上端のドラッグハンドル（`BottomResizeBar`）で可変。**概要パネル（左）はこの排他と無関係**で、`resized()` では下部→左の順に領域を取り、詳細に横幅フルを与える
- **ピアノロールの閉じる導線**: ✕ボタン＋`onCloseRequested`。FXパネル・詳細も同じパターンにする
- **ON/OFFフラグの置き場所は `TrackParams` のatomic**（`eqEnabled`/`compEnabled`、デフォルトtrue）: スライス3以降でオーディオスレッドが直接読む値なので、gain/pan/sendsと同じ共有atomicパターンに乗せる（スナップショット再構築不要でON/OFF即反映）。保存は各トラックに `"fx": { "eq": {"enabled": true}, "comp": {"enabled": true} }`（後続スライスでパラメータを同じオブジェクトに足せる形）
- **ミキサー→エディタのチャンネル切替**: `MixerOverlay` のバス/Masterストリップは現状クリック無配線（`MixerStrip::onSelect` はトラックのみ）。バス/Master用の `onSelect` を配線し、MainComponent経由でエディタの表示対象（enum: 選択トラック追従 / バスb / Master）を切り替える。ミキサーの `dismiss()` 時に追従へ戻すため、閉じイベントの通知が要る（keyPressed の X/Esc・パネル外クリックの3経路すべてで）
- **sendノブの二重表示の同期**: ミキサーと概要パネルが同時に見えるとき、同じ `TrackParams` を2つのUIが表示する。値の書き込みはatomic直書きで衝突しないが、**表示の同期が要る**: パネル側は `onSendChanged` / `onFxEnabledChanged` を分け、sendのみミキサーの `refreshValues()` を呼ぶ（ON/OFFはミキサーに表示がないためdirty化のみ）
- **ノブ・ON/OFF豆ボタンの部品はスライス1の実装を流用**: ロータリー描画は `AppLookAndFeel::drawRotarySlider`（済）、豆ボタンは flatButton スタイル、sendノブのドラッグ中値表示は `MixerOverlay.cpp` の実装をコンポーネント化して共有する（コピペで2箇所に増やさない）
- **`I` キーは未使用**（Shortcuts.h確認済み）。テーブルに追加すれば⌘?一覧は自動反映
- **トラック削除・undo等でモデルが変わったときの防御**: パネルの表示対象トラックが消えたら選択トラック追従に戻す（`refreshFromModel`）。内部同期の `selectTrack()` はバス/Master表示を維持し、ユーザーのトラック選択（`selectTrackFromUser()`）だけが追従を切り替える。詳細は `syncFxDetail()` で追従（トラック→トラックは同スロット・構成が変わる切替は閉じる）

### 方針変更

- **下部エディタ→左ペイン常設に変更（2026-07-23・ユーザー実機確認後の判断）**: 動くものを見た結果「Logicのインスペクタのようにトラック一覧の左に常設したい」となった。キーは `B`→`I`（Logicのインスペクタ準拠）、プロジェクトを開くと表示済み
- **さらに「左=概要＋下部=詳細」の2段構成へ（同日）**: ユーザーの元々のイメージは「Logicで言うフローティングの詳細画面を下部に出す」だった。左パネルはスロット一覧＋Sendsの概要に縮め、スロットクリックで下部に詳細エディタ（横幅フル）を開く形に再構成。これでEQカーブの横幅問題（左ペイン240px制約）も解消。詳細とピアノロールは下部スロットを排他（後勝ち）だが、概要パネルは常に見えるので実害は小さい
