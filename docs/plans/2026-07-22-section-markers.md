# セクションマーカー（アレンジメントマーカー）

## 概要・やりたいこと

Logic Proのアレンジメントトラックのように、小節ルーラーの下に「intro / verse / hook ...」のセクションラベル帯を表示し、曲のどこがどのセクションかを一目でわかるようにする。あわせてセクション頭へのシーク（クリック）をボーカル録り直しの動線として使えるようにする。

- Logicと違いラベルは自由入力にせず**6種の固定選択式**、色も**種別ごとに固定**（選ぶ認知コストをゼロにする「引き算」方針）
- マーカーはただのラベルであり、リージョン・クリップには一切触らない（Logicの「挿入時に後ろをずらす」編曲機能はスコープ外）

## 前提・わかっていること

### 設計の決定事項（相談で確定）

- **区間方式（Logicのアレンジメントマーカーと同じ）**: マーカーは `{開始小節, 種別}` のみ持ち、終了位置は持たない。自分の終わり = 次のマーカーの開始（最後は曲末まで）。重なり・隙間・リサイズの端数が構造上存在しない
- 最初のマーカーより前の範囲は無ラベルの空白でよい（全小節カバーの義務なし）
- **種別は6種固定**: `intro / verse / hook / bridge / outro / other`。将来増やすときはenumに1行足す
- **色は種別のプロパティ**（ユーザー選択なし）。5種は有彩色、`other` は無彩色グレー（「特定の種別ではない」ことが一目でわかる）
- **番号は自動採番（Logic式）**: 同種別が2個以上あれば出現順に `verse1, verse2...` と表示、1個だけなら番号なしで `verse`。番号はデータに持たず表示時に計算（削除時の繰り上がりが自動で正しくなる）
- **開始位置は拍単位（上限）**: スナップは表示中グリッド準拠で、ズームアウト中は小節頭・ズームインすると最細で拍。
  拍より細かくはしない（2/4小節が挟まる曲の「半小節ずれ」に追従するのが目的で、1/16等は誤操作リスクにしかならない）

### 操作仕様（相談で確定）

| 場所 | 左クリック | 右クリック |
|---|---|---|
| レーンの空白 | 種別メニュー → その小節頭から追加 | 同じ |
| 既存セクション上 | **クリック=セクション頭へシーク / ドラッグ=開始小節の移動** | ここにセクションを追加（=分割）/ 種別変更 / 削除 |

- 「間に追加」= 既存セクションの途中に境界を1本立てる分割。例: `verse(9-24)` の bar 17 で右クリック→bridge追加 → `verse(9-16) / bridge(17-24)`
- 削除 = 境界が1本消え、前のセクションがそこまで伸びる
- ドラッグ = 開始小節の移動（本体=掴んだ位置からの相対移動・左端±4px=カーソル位置へ吸着）。隣のマーカーは越えられない。
  シークは「動かさず離したクリック」のみ（mouseUp判定。移動量が1小節未満ならクリック扱いでシーク）
- 同じ小節頭に2つのマーカーは置けない（既にマーカーがある小節への追加は種別変更として扱う）

### コードの現状（調査済み）

- ルーラーは `TimelineView` 内の `RulerContent`（`rulerHeight = 26` 固定、`Source/ui/TimelineView.cpp:30`）。横スクロールは `ruler->setTopLeftPosition(-viewport->getViewPositionX(), 0)` で同期（`syncScroll`）
- レイアウトは `resized()` で `viewport->setBounds(0, rulerHeight, ...)`。マーカーレーンを挟むならここと `syncScroll` に1本足す
- モデルは `Source/shared/Project.h` の `Project`。`save`/`load` は `Project.cpp` の DynamicObject ベースで、フィールド追加は素直にできる。マーカーは再生に関与しないので `PlaybackSnapshot` への追加は**不要**
- undo は既存の `onWillEditModel`（編集直前スナップショット）/ `onModelEdited`（確定通知）フックに乗せる。ただし **`UndoStack`（`Source/shared/UndoStack.h`）は `Project::tracks` しか保存・復元しない**ため、保存対象に `markers` を含める拡張が必要（レビュー指摘）
- **`updateContentSize()`（`TimelineView.cpp:483`）はクリップ・MIDIリージョン・再生位置からしかコンテンツ幅を出さない**。後方のマーカー（例: bar 100）が幅計算に入らないと表示も操作もできないため、最大位置の算出にマーカーを加える必要がある（レビュー指摘）
- **小節番号は1始まり、タイムライン内部座標は0始まり**。`x = (startBar - 1) * pxPerBar`、`sample = (startBar - 1) * barLengthSamples` の変換を1箇所に共通化し、追加・描画・シークでの1小節ずれを構造的に防ぐ（レビュー指摘）
- ショートカット追加時は `Source/ui/Shortcuts.h` のテーブルが単一の真実の源（CLAUDE.mdのルール）。`,`/`.` = 1拍、`Shift+,`/`.` = 1小節が既存
- テスト基盤あり: `Tests/TestsMain.cpp`（CTest登録済み・`ctest` で実行）。Project の save/load ラウンドトリップと `testUndoStack()` が既にあり、マーカーのテストはこの流儀に合わせて足す
- project.json のバージョンは v3。マーカーは無くても読める任意フィールドとして足すので**バージョンbump不要**（旧ファイルは `markers` 無し = 空として読む）

### 方針変更

- UI経由のundo/redo確認は合成キー不可（JUCE既知の制約）のため人間確認に移動した
- 移動操作を「境界±4pxのみドラッグ可」から「**本体ドラッグでも移動可**（相対移動・Logicと同じ）」に変更。
  ±4pxは実操作では狭すぎた（ユーザー指摘）。シークは「動かさず離したクリック」に限定して両立
  （mouseDown即シーク → mouseUp判定シークに変更）
- 配置粒度を小節頭固定から**拍単位（表示グリッド準拠・上限=拍）**に拡張（ユーザー要望: 2/4小節が挟まる曲の
  半小節ずれに遭遇するため）。モデルは `startBar:int` → `startBeats:int`（曲頭からの拍数）に変更。
  JSONは `bar`+`beat`（省略時0）で後方互換のため既存プロジェクトの移行不要。実機で bar6 beat2 への
  分割挿入→保存→JSONの `beat:2` を確認済み
