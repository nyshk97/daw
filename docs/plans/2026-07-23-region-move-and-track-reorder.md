# リージョン移動のトラック跨ぎ対応・オーディオクリップのドラッグ移動・トラック並び替え

## 概要・やりたいこと

Logic Pro でできる以下の3操作を LaLa に実装する。

1. **オーディオリージョン（クリップ）のドラッグ移動**: 現状は選択と右クリックメニューのみ。横方向（時間軸）＋トラック跨ぎの移動を追加する
2. **MIDIリージョンのトラック跨ぎ移動**: 横移動・リサイズ・⌥複製は実装済みだが、縦方向（別トラックへの移動）が未対応
3. **トラックの並び替え**: Logic 同様、トラックヘッダのドラッグ＆ドロップで並び替える

## 前提・わかっていること

### 深掘りフェーズでの決定事項
- オーディオリージョンの移動は**トラック跨ぎも含む**（MIDI用に作る跨ぎ機構を流用）
- トラック跨ぎは**同種トラック間のみ**（audio→audio、midi→midi）。audio↔MIDI 間は型が違うので不可
- トラック並び替えは**ヘッダのドラッグ＆ドロップ**＋挿入位置インジケータ
- グリッドスナップ・undo（`onWillEditModel`）・再生中の反映（snapshot差し替え）は既存パターンを踏襲
- 再生への反映は**ドロップ（mouseUp）時のみ**（既存MIDIドラッグと同じ。ドラッグ中のリアルタイム反映はしない）
- 無効レーン（異種トラック・余白・境界外）上では**最後に有効だったトラックに残す**（ドラッグ開始元へ戻さない）
- **録音中は並び替え不可**（トラック削除の既存ガードと同じ方針。`pendingRecordTrack` の index 保持を維持できる）
- 並び替え時の index 参照（選択トラック・Timeline両選択・FXエディタ・ミキサー）は **`Track::id` 退避 → 新 index 引き直し**で追従させる

### コードベースの現状（調査済み）
- データモデルは `Source/shared/Project.h`。`Clip`（`startSample` 絶対サンプル位置、`audio` は `shared_ptr` 共有の非破壊参照モデル）、`MidiRegion`（`startPpq` 絶対PPQ）、`Track`（`clips` / `midiRegions` の vector を直接保持、永続ID `Track::id` あり）
- **表示順・保存順・再生順 = `project->tracks` の vector 順**。order フィールドは無い。並び替えは vector の入れ替えだけで一貫する
- MIDIリージョンのドラッグは `Source/ui/TimelineView.cpp` の `RegionDrag`（`TimelineView.h:138-148`、Mode: none/move/resize）＋ `handleLaneMouseDown/Drag/Up`（`:704-`, `:804-`）で実装済み。最初に動いた時だけ `onWillEditModel`（undoスナップショット）、グリッドスナップあり
- オーディオクリップの `hitTestClip`（`TimelineView.cpp:926`）は選択・右クリックメニューにのみ使用。`clip.startSample` を書き換えるUIは存在しない
- 再生エンジンはロックフリーの snapshot 差し替え（`SnapshotExchange`）で編集を反映。位置変更は `buildSnapshot` → `push` で再生中でも安全
- `PlaybackEngine` は重なったクリップを単純に加算ミックスする → **移動でクリップが重なっても特別処理はしない**（重なりを許容。Logic のような上書き・クロスフェードは作らない）
- Tier2プランに「トラック削除・並び替え等はインデックスでなくIDで扱う」の設計方針が明記済み
- Tier1プランでオーディオクリップの移動・並び替えは明示的に見送られていた（今回それを解禁する位置づけ）

### Phase 2: トラック跨ぎ移動（MIDI・オーディオ共通） [AI🤖]
- [x] **編集開始判定を縦方向にも拡張する**: 現行の `handleLaneMouseDrag` は横方向の差分がゼロだと `onWillEditModel` の前に早期 return する（`TimelineView.cpp:817` の `if (deltaPpq == 0) return;`）。このままでは時間位置を変えずに別トラックへドラッグしたとき跨ぎ移動も⌥複製も始まらない。編集開始条件は次の両方を満たすこと:
  - ドラッグ閾値を超えている
  - かつ、**スナップ後の時間位置が変わる、または有効なターゲットトラックが現在所属トラックから変わる**

  無効レーン上や同一レーン内で縦にぶれただけではモデルを変更せず、⌥複製・undo登録（`onWillEditModel`）・dirty化も発生させない。編集開始が成立した最初の1回だけ `onWillEditModel` を呼ぶ（横差分ゼロのトラック跨ぎでも呼ばれる）
- [x] ドラッグ中の Y 座標からターゲットレーン（トラック）を判定するヘルパーを追加
- [x] 跨ぎ判定: 同種トラック（audio→audio、midi→midi）のみ許可
- [x] **無効レーン上の挙動を固定**: 無効な位置（異種トラックのレーン・トラックの無い余白・コンポーネント境界外・スクロール中に有効レーンを見失った場合）ではモデルを変更せず、**最後に有効だったトラック**に所属・表示したままにする（ドラッグ開始元へは戻さない）。横位置（時間軸）の更新は無効レーン上でも継続する
- [x] レーン境界を越えた時点でリージョンを元トラックの vector から erase し、ターゲットの vector へ移動（ライブ移動方式。ドラッグ状態のトラック/リージョンインデックスを追従更新）。フローティング描画の別経路は作らず、モデル移動＝描画反映で完結させる
- [x] undo は最初の移動時の `onWillEditModel` スナップショットで跨ぎ含め丸ごと復元されることを確認
- [x] MIDIリージョンの跨ぎ後は移動先トラックの音源（gmProgram/drums）で発音される（Logicと同じ。リージョン側は notes を持つだけなので追加実装なしのはず。確認のみ）
- [x] 選択状態・ドラッグ中インデックスの整合を確認（vector 間移動でインデックスがずれる箇所を潰す）

### Phase 3: トラックヘッダのドラッグ並び替え [AI🤖]
- [x] **録音中は並び替え不可**とする。ただし `TrackHeaderComponent` は engine への参照を持たない（既存も `onSelect` / `onDeleteClicked` 等のコールバック経由）ため、**ヘッダへ `std::function<bool()> canReorder` コールバックを渡し、MainComponent 側が `! engine.isRecording()` を返す**構造にする。ドラッグ開始（閾値超え）の時点で判定してブロック（トラック削除の既存ガード `MainComponent::requestDeleteTrack` の `MainComponent.cpp:608` と同じ方針）。これにより `pendingRecordTrack` は index のままでよい（録音中に順序が変わらないため、停止時のクリップ追加先がずれる事故は起きない）
- [x] `TrackHeaderComponent` の mouseDown/mouseDrag/mouseUp にドラッグ処理を追加（数pxの閾値超えで並び替えモード開始。既存のクリック選択・削除メニューと共存させる）
  - **ドラッグ開始可能領域はヘッダ背景＋トラック種別アイコン領域のみ**（子コンポーネント＝nameLabel・M/Sボタン・音量スライダー・楽器セレクタの上では開始しない。JUCEの子コンポーネントがマウスを奪うため自然にそうなるが、仕様として固定する）。実装後、背景の掴み面積が狭すぎる場合は nameLabel からのドラッグ転送を検討し、その際はリネーム操作（ダブルクリック等）と競合しないことを確認する
- [x] ドラッグ中、`TrackHeadersView` に挿入位置インジケータ（ヘッダ境界の横線）を描画
- [x] mouseUp で `project->tracks` を並べ替え（`std::rotate`）。`onWillEditModel` → 並べ替え → **index参照の引き直し（下記）** → `buildSnapshot` → `push`
- [x] **並び替え後に誤ったトラックを参照しないよう、`Track::id` 基準で index を引き直す**。手順: 並べ替え前に以下の各参照が指すトラックの `Track::id` を退避 → vector 入れ替え → id から新 index を解決して書き戻す。対象（いずれも現状 index 保持）:
  - `MainComponent::selectedTrack`（`MainComponent.h:157`）→ 引き直し後 `headers.setSelectedTrack` / `timeline.setSelectedTrack` で再同期
  - `TimelineView::ClipSelection::track` / `RegionSelection::track`（`TimelineView.h:16-30`。選択中リージョンが別トラックを指すのを防ぐ）
  - `FxEditorView::targetTrack`（`FxEditorView.h:66`。開いているFX詳細エディタが別トラックの params を表示・編集するのを防ぐ）
  - 開いているミキサー（`mixerOverlay`。`sync`/`showOver` が selectedTrack 起点なので、selectedTrack 引き直し後に `mixerOverlay.sync` を呼び直す）
  - 上記以外にも index を跨いで保持する箇所がないか grep（`tracks[` / `trackIndex` / `selectedTrack`）で最終確認する
- [x] タイムライン側のレーン表示・FXパネル等、トラック順に依存するUIが並び替え直後に再構築されることを確認
